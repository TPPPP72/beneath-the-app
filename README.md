# 计算机系统演化

> **微架构、操作系统、编译器与异构计算如何共同塑造现代计算**

一本以**系统演化**为主线的开源计算机书籍。

本书试图回答一个贯穿整个计算机发展史的问题：

> **现代计算机为什么会演化成今天这样？**

与按照 CPU、操作系统、编译器、网络等领域分别介绍知识的传统教材不同，本书以时间为主线，从计算理论开始，追踪微架构、操作系统、编译器、存储系统、GPU 与异构计算之间相互推动、共同演进的全过程。

---

# 本书希望回答的问题

为什么会有操作系统？

为什么 CPU 必须支持中断？

为什么高级语言最终取代了汇编？

为什么会出现 Cache？

为什么处理器开始采用流水线、乱序执行和分支预测？

为什么单核性能最终停止增长？

为什么 GPU 能够成为 AI 计算的核心？

为什么今天又开始重新研究存内计算、Chiplet 与异构计算？

这些技术并不是孤立出现的。

它们都是为了**解决上一代计算机留下来的系统瓶颈**。

本书希望按照历史发展的顺序，将这些演化过程串联起来。

# 本书特色

## 从"为什么"开始，而不是"是什么"

除总结章节外，每一章都遵循统一的行文主线：

```text
背景问题（Problem）
        │
        ▼
解决方案（Solution）
        │
        ▼
系统层级改变（New Layer）
        │
        ▼
新的问题（New Problem）
        │
        ▼
历史影响（Historical Impact）
```

重点关注：

- 为什么会出现新的抽象？
- 为什么旧方案无法继续发展？
- 软件如何推动硬件？
- 硬件又如何催生新的软件？

---

## 强调系统协同演化

现代计算机从来不是某一个领域单独发展的结果。

例如：

```text
      晶体管
        │
        ▼
     操作系统

     操作系统
        │
        ▼
    MMU 与特权级

     高级语言
        │
        ▼
复杂 ISA 与编译器优化

      Cache
        │
        ▼
    现代微架构

       GPU
        │
        ▼
   CUDA 与 GPGPU

       AI
        │
        ▼
     异构计算
```

软件不断提出新的需求，而硬件不断提供新的能力，两者共同推动了现代计算机的发展。
